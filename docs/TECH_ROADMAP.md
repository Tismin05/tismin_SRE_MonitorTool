## 三、阶段 2：数据存储 + 历史趋势

### 3.1 技术栈选择

| 组件 | 技术 | 版本 | 说明 |
|------|------|------|------|
| 时序数据库 | **InfluxDB** | v2.7.x | 专为监控指标设计 |
| 或 | **Prometheus** | v2.48.x | 监控领域标准，支持 Pull 模式 |
| 客户端 | **influxdb-client-go** | v2.12.x | 官方 Go SDK |

**选择建议**：
- **InfluxDB**：适合自己采集、自己存储、自己查询的场景
- **Prometheus**：适合服务暴露接口、Prometheus 定期拉取的场景

### 3.3 InfluxDB 数据模型

```
Measurement: system_metrics
Tags:
  - host: server01
  - type: cpu|memory|disk|net

Fields:
  - usage_percent: float
  - total: uint64
  - used: uint64

示例查询：
from(bucket: "tisminSRETool")
  |> range(start: -1h)
  |> filter(fn: (r) => r._measurement == "system_metrics")
  |> filter(fn: (r) => r._field == "cpu_usage_percent")
```

## 六、技术选择总结

### 推荐组合

```
核心路线：
┌─────────────────────────────────────────────────────────────────┐
│  Go Backend (Gin + Viper)                                       │
│       │                                                         │
│       ├─▶ InfluxDB (时序存储)                                   │
│       │                                                         │
│       └─▶ Vue 3 + Element Plus + ECharts (前端)                 │
└─────────────────────────────────────────────────────────────────┘

备选/扩展：
- 日志: zap
- 实时: WebSocket
- 监控: 对接 Prometheus + Grafana
```

### 为什么选择这些技术？

| 技术 | 选择理由 |
|------|---------|
| **Gin** | Go 最流行的 HTTP 框架，性能好，文档完善 |
| **Viper** | Go 配置管理的事实标准 |
| **InfluxDB** | 专为时序数据设计，查询语法 Flux 强大 |
| **Vue 3** | 学习曲线平缓，Composition API 灵活 |
| **Element Plus** | Vue 3 生态最完整的 UI 库 |
| **ECharts** | 图表功能强大，支持大数据量渲染 |
